網誌搬家！改用 Pelican + GitHub Pages

by jsliang
in Blog on 2013-02-15 | tags: Python Pelican GitHub GitHub Pages Sublime Text | comments

約莫 2011 年底的時候，看到 Octopress 非常地流行，一度手癢想把網誌從 Wordpress 換成用 Octopress。不過那時一來研究所比較忙，二來我對 Ruby 不熟，所以一直沒興趣動手。

最近我的 AWS EC2 變得怪怪的。雖然立即將原本 Wordpress 上的資料備份，但是總是有點怕出意外，不太敢在上面再寫新文章。為此我重新考慮使用靜態網站產生器。

為何要用靜態網站產生器？

其他人的理由

Tzeng Yuxio 有整理了一系列 Octopress 開箱文，列舉大家換到 Octopress 後發的第一篇文。

大部份的人都是從動態管理系統如 Wordpress、Blogger 轉過來的，因此文中通常也會說明自己為何會想嘗試像 Octopress 這樣的靜態網站產生器，我這裡就先不贅述了 。

我的理由

程式碼排版方便、可以用 Markdown 寫作等特色對我而言的吸引力都還好，想搬家的原因主要是 hosting 與備份很容易。

從我的 AWS EC2 變得怪怪的時候開始，我意識到我需要一個可以很容易儲存、備份我的文章的方式。最直覺想到的做法是透過 Dropbox。然而 Wordpress 要備份的除了文章內容，也包括佈景主題、套件等。雖然有很多套件在處理這一塊，但我總是覺得很麻煩，生怕自己遺漏了什麼。

此時我想到最單純的做法就是把文章寫在檔案裡，用程式處理後轉成靜態網頁，然後放在 GitHub Pages 上。如此一來原始文章以及產生的靜態網頁都是純文字檔案，便可以很容易用 Dropbox 管理。上傳到 GitHub 更是再多了一個備份地點。

而這些正是 Octopress 能做到的事。只是近來我對 Python 比較有興趣，於是上網搜尋了一下是否有 Python 程式可以達到一樣的目的。接著發現 Pelican 文件頗齊全、GitHub 上也很活躍地持續有新開發，因此決定來嘗試 Pelican。年假期間花了約兩天，把文章從 Wordpress 都移到 Pelican 了。

安裝 Pelican

下載及安裝可參考 Pelican 的 v3.1.1 官方文件: Getting started。

按照文件中的步驟做，就可以跑起來一個很簡單的網誌了。痛苦的地方從舊文匯入開始……

把 WordPress 的文章轉成 Markdown 格式

官方的 pelican-importer 支援 WordPress、Dotclear、RSS/Atom feed 的匯入。

然而我用 pelican-importer 匯入我的 Wordpress 文章時，遇到了兩個比較大的問題。

  1. 在 Windows 下使用時，若原始文章有中文字元會出錯。後來改在 Ubuntu 12.04 的環境下執行就沒問題。
  2. 匯出格式是 reStructuredText，而 reStructuredText 似乎沒辦法表達用圖片做超連結如 <a href="..."><img src="..."/></a> 這樣的情況。

第二點令我很頭痛，因為有陣子我的文章中很常把圖片拿來用作超連結。Markdown 可以正確處理，因此我改用 wp2md 來做轉換。不過 wp2md 轉完問題也不少……

  1. wp2md 使用的 metadata 欄位名稱和 Pelican 的慣用名稱不同，例如 Pelican 用的是「date」，而 wp2md 轉出來的欄位名稱用的是「post_date」…等。

    所幸這問題不大，挑個好一點的編輯器如 Notepad++ 或 Sublime Text 一次針對多個檔案做字串取代即可。

  2. wp2md 沒有處理 category。

    這蠻囧的。這部份可以看是要去改 wp2md 的 code，或是自己寫 script 和 pelican-importer 的輸出檔來比對以抓出各篇文章的 categories。我是採取後者的做法。

  3. wp2md 沒有把 <br> 轉成斷行。

    這超級囧的，反而是 pelican-importer 的輸出處理得比較對。

    我以前寫文章很常有 <p>第一行<br>第二行<br>第三行</p> 這樣的寫法，在 wp2md 的輸出會變成 <p>第一行 第二行 第三行</p>。慘的是我不能一味把所有的空白取代成斷行，因為有些真的本來就是空白……

    解法可以看是要去改 wp2md 的 code，或是自己重新一篇篇校稿。因為文章不多（不到兩百篇），所以我是採取後者的做法。後來我才發現這決定是個錯誤。

為了處理文章轉換，我今年過年真的守歲了。本想說我文章不多，一篇篇整理應該也不致於花太多時間。不料我就這樣從除夕晚上做到大年初一早上，看到日出才去睡。意外收穫是我邊處理邊把「甄嬛傳」放在背景播，一連聽了十多集的台詞，進度追得超快呢。

設定 nginx 轉址

此次搬家我連網域也順便換了，因此必須設定轉址。

設定方式我寫在這篇文章：轉移網域後，在 nginx 設定 301 Moved Permanently

把版面改成 responsive layout

Pelican 預設的佈景主題使用的是 Smashing Magazine 的設計。其實我很喜歡這個版面，但它不是 responsive web design，在現在行動裝置稱霸的時代有點可惜。

因此我暫且很沒創意地使用 Twitter Bootstrap 的 framework，製作了一個佈景主題 Fresh（也就是網誌現在在用的這個佈景主題）。留待未來行有餘力再來美化佈景主題了。

使用 Sublime Text 2 寫 Pelican 文章

最近迷上了 Sublime Text 2 這個編輯器，很喜歡拿它來寫 code。

為了方便用 Sublime Text 2 寫 Pelican 的文章，我寫了個套件 SublimePelican。目前有支援自動插入文章 metadata 的功能（包含自動產生文章日期和文章的 slug）。

至於 Sublime Text 2 在 Linux 環境下無法輸入中文的問題，我是用 InputHelper 解決。

其他……

這篇文章只是很隨性地記了些我在轉換過程中遇到的問題，待我多寫了幾篇文章後再來寫使用心得。:P
