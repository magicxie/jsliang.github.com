WordPress: 用SQL語法對特定category/tag的所有文章一次新增其他category/tag

2011-02-02 17:04:51
By jsliang

In Blog.

我知道文章標題看起來很讓人困惑，但我國文不好，這功能我實在不知道要怎麼用文字描述得更清楚了……

以我的需求為例。我的文章常常放照片，而我習慣在文章標記拍攝用相機的型號。

之前考慮到同個廠牌有不同型號的相機，有可能未來會想要一次瀏覽某廠牌的相機所拍的照片，因此我會把廠牌和型號分成兩個tags。 例下所示：

Canon EOS 500D = Canon + EOS 500D

Pentax Optio A36 = Pentax + Optio A36

Olympus XZ-1 = Olympus + XZ-1

但我今天不知道哪根荕不對，想要分別再加上「Canon EOS 500D」、「Pentax Optio A36」、「Olympus XZ-1」的tags……

用這篇文章記錄一下我的做法。

當然使用WP的管理介面也可以做到這樣的事，但我操作起來卻不太順。

不知道為什麼，我每次用WP管理介面的批次文章管理功能，處理速度總是很慢，還常常傳回錯誤訊息，非常考驗我的耐性……我尚未弄清楚到底是我的機器的問題，還是WP管理介面的程式的問題。

以下的做法，必須是自己架設WordPress程式，並且可以連到WP的MySQL資料庫下SQL語法才有用。

我想做的事情：把每篇有「XZ-1」tag的文章都再新增一個「Olympus XZ-1」的tag。

  1. 確定category/tag的名稱。 如果是之前沒用過的，到 WP管理介面 > Posts > Categories / Post Tags 新增你想加的category或tag，並記住你所使用的名稱。 以我的例子，就是在Post Tags新增一個名稱為「Olympus XZ-1」的tag，我要記住的名稱就是「Olympus XZ-1」。

  2. 登入WP的MySQL資料庫，執行以下SQL語法：

    INSERT INTO wp_term_relationships SELECT A.object_id, NEW.term_id AS term_taxonomy_id, A.term_order FROM wp_term_relationships AS A, wp_terms AS REF, wp_terms AS NEW WHERE A.term_taxonomy_id = REF.term_id AND REF.name = 'XZ-1' --參考的category/tag的名稱 AND NEW.name = 'Olympus XZ-1' --要新增的category/tag的名稱

請自行修改上面程式碼的「XZ-1」與「Olympus XZ-1」。

  1. 新增完成！

實驗心得：自己下SQL語法真的是快多了……只是有點可惜了WP的精美管理介面。Orz
